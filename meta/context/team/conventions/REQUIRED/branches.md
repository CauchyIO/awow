# Branches

awow uses GitHub Issues + Projects, so branch names are not auto-generated by the board tool. Pick a pattern that maps a branch to its issue at a glance.

## Pattern

| Context | Pattern | Example |
|---|---|---|
| Issue-linked work | `{user}/issue-{number}-{short-slug}` | `casper/issue-42-gh-cli-as-mcp-alternative` |
| Long-lived feature | `feature/{descriptive-slug}` | `feature/dry_run_awow` (the current working branch) |
| Hotfix on `main` | `hotfix/{short-slug}` | `hotfix/gather-skips-symlinks` |
| Maintainer prompt iteration without a board issue | `prompt/{short-slug}` | `prompt/setup-awow-step1-tightening` |

`{user}` is the GitHub login. `{short-slug}` is kebab-case, ≤ 6 words, drawn from the issue title.

## Rules

1. **One branch, one issue.** When a branch crosses scope, open a second issue and a second branch.
2. **Do not push to `main` directly** — every change goes through a PR, even single-line fixes. The wizard relies on PR review as the human checkpoint.
3. **Delete the branch on merge.** GitHub's auto-delete-on-merge setting is on for `CauchyIO/awow`; do not push stale branches back after a merge.
4. **Long-lived branches get a tracking issue.** A `feature/*` branch without a tracking issue is invisible to the wizard's status reports.
