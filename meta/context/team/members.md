## Growth

When a contributor joins:

1. Open an issue titled `Add {name} to dogfood members.md` (per `conventions/REQUIRED/issue-titles.md`).
2. Add a section above "Growth" with their role / responsibilities / focus / handle.
3. Update CLAUDE.md (regenerate via `tools/bootstrap-claude-md.py`) so the agent sees the new context next session.

A one-person team is a useful constraint, not a problem to hide — the conventions are easier to evolve when there's exactly one source of taste.
