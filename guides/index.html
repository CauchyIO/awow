Why awow exists

Above a certain size, every organisation develops a fog: the question what is actually happening right now? gets three or four answers depending on who you ask. awow clears it by making the board the single source of truth and giving an agent the context to keep it honest. The guides here are how a team gets there — one phase at a time.

01

Setuplay the foundation

An agent is only as useful as the context you give it. Most of the value comes from writing that context down clearly, once, in the right files — and from wiring the board so the agent can read and write it. Setup is the /setup-awow wizard plus the board surface it configures.

02

Seedthe everyday loop

If the board is where the surgery happens, the Seed loop is the surgery. Ceremonies get sharper once the artefacts they depend on are accurate: refinement that prepares itself, stories sized so a single session ships a working PR, a daily reflection that keeps the board honest. This is the loop one person runs that proves the value.

The thinking behind it

Where these guides come from

The guides are the hands-on layer. The reasoning underneath — the coordination fog, the board-as-graph reframe, ceremonies rethought, the prompt landscape, governance and works-council considerations, and a worked rollout — lives in the published Agentic Way of Working methodology and its Boskalis case study. Each phase above distils the idea you need to use the guides; the full argument is the long read.

One folder, four phases. You don't have to adopt them in order all at once — most teams live in Setup and Seed for a while before reaching for Spread and Standardise. The roadmap is a map, not a checklist.

Source of truth for every command and skill referenced here: .agents/commands/ and .agents/skills/. This index and all four new guides are self-contained — no external fonts, links, or scripts; links point only to the sibling guide files in this folder.
